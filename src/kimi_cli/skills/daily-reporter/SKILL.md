# daily-reporter

用于生成工作日报的 Skill。输出为简洁的 Markdown 列表格式，不含 emoji 图标。

## 触发条件

用户说以下任意一种表达时触发：
- "写日报"
- "生成日报"
- "今天日报"
- "日报"
- "帮我写一下今天的工作日报"

## 写作 SOP（必须严格执行）

1. **读取项目输出规则**：先读取 `/Users/qunwei/Documents/local_agent_work/output/README.md`，确认日报目录与命名规则。
2. **确认当前日期**：运行 `date +%F` 获取真实日期，绝不依赖会话上下文或文件修改时间。
3. **扫描目录**：列出 `output/reports/daily/<月份>/` 目录，查找是否已有 `D-YYYY-MM-DD.md` 或 `YYYY-MM-DD_日报.md`。
4. **复用已有文件**：如果当天文件已存在，在其末尾追加新内容；不要创建第二个文件。
5. **命名**：默认使用 `D-YYYY-MM-DD.md`；若 README 另有规定，从其规定。
6. **写完后校验**：用 `ls -l` 和 `head` 确认文件路径与文件内日期一致。
7. **回复时报告**：报告日报的绝对路径与校验结果。

## 日报模板

```markdown
### YYYY-MM-DD 工作日报

**今日核心成果**
- 关键成果 1
- 关键成果 2

**项目进展**
- 项目 A：完成了什么
- 项目 B：推进到哪一步

**问题与对策**
- 问题 1：描述 → 对策
- 问题 2：描述 → 对策

**明日计划**
- 计划 1
- 计划 2

---
*路径：<日报绝对路径>*
```

## 内容要求

- 用简洁的条目写，不展开长段落。
- 优先记录「形成产品形态」类的实质性进展。
- 问题与对策成对出现，不写只有问题没有对策的条目。
- 不要添加任何 emoji、图标或装饰性符号。
- 如果某项工作无实质进展，可直接省略对应小节。

## 输出位置

默认保存到 `/Users/qunwei/Documents/local_agent_work/output/reports/daily/<月份>/`。
