# docx-media-aware

## Description

处理任何 Microsoft Word `.docx` 输入任务时，必须先识别并处理文档中的图片、图注、截图和嵌入式媒体，再执行主任务。适用于翻译审校、文档提取、内容总结、格式转换、合规审查等任何涉及 Word 文档的工作流。

## Trigger

当用户任务满足以下任一条件时，**必须**先调用本 skill：

- 输入文件包含 `.docx` / `.doc` / Word 文档；
- 任务涉及“审校翻译”、“翻译 review”、“提取文档内容”、“总结文档”、“转换格式”、“文档对比”；
- 用户提到“Word 里有图片”、“文档里有截图”、“图注也要看”等。

## Why this exists

过去在处理 Word 文档任务时，默认只读取纯文本，导致：

- 翻译审校漏掉图注（figure captions）里的英文错误；
- 忽略截图中的 UI 文字、流程图、标注；
- 不知道文档里嵌入了多少张图片，遗漏视觉内容。

本 skill 强制要求：**在主任务开始之前，先完成图片/媒体审计**。

## Workflow

### Step 1: 扫描文档中的媒体

对每一个输入的 `.docx` 文件，运行：

```bash
python /Users/qunwei/Documents/local_agent_work/skill/docx-media-aware/extract_docx_media.py \
  <input.docx> \
  --out-dir /Users/qunwei/Documents/local_agent_work/output/docx_media/<task_name>/
```

该脚本会输出：

- `media/`：提取出的所有图片文件（png/jpg/emf/svg 等）；
- `media_index.json`：每张图片的文件名、原始 docx 中的 rId、出现顺序、附近的段落文本、疑似图注；
- `summary.txt`：媒体总数、类型分布、是否有图注被识别。

### Step 2: 判断是否需要处理图片

根据 `summary.txt` 的结果：

| 情况 | 动作 |
|------|------|
| 无图片 | 直接开始主任务，但需在汇报里说明“本文件未检测到嵌入式图片” |
| 有图片，且任务是翻译审校/文档提取 | **必须**至少抽样查看关键图片（截图、流程图、带文字的图片），并把图注/截图文字纳入审校范围 |
| 有图片，但用户明确说“只看文字” | 向用户确认后，仅处理文字，并记录此例外 |
| 图片 ≤ 20 张 | 建议全部读取 |
| 图片 > 20 张 | 先向用户汇报媒体审计结果，按“图注/截图/流程图优先”策略抽样读取；若用户要求全读再扩大范围 |

### Step 3: 读取图片内容

- 对 `media/` 中的图片使用 `ReadMediaFile` 工具或视觉能力进行查看；
- 对截图类图片（UI、界面、表格截图）要识别其中的文字和布局；
- 对流程图/架构图要理解其结构和标注；
- 对照片类图片判断是否与主任务相关。

### Step 4: 把图片信息写入上下文

在主任务提示词里必须包含：

- 图片总数；
- 关键图片的编号和简短描述；
- 哪些图片包含文字/术语/图注需要审校；
- 提取出的图片保存路径。

### Step 5: 执行主任务

执行翻译审校、提取、总结等主任务时，必须同时参考：

- 文档文字内容；
- 图片内容；
- 图注和附近的上下文。

## Rules

1. **禁止**在未检查图片的情况下直接说“我已经读完文档”。
2. **禁止**仅使用 `python-docx` 默认文本提取后就开始主任务；`python-docx` 不会输出图片。
3. 如果图片是截图且包含英文文字，翻译审校必须覆盖这些文字。
4. 图注（Figure X ... / 图 X ...）与图片必须成对检查；发现图注错误或图片与图注不一致要单独列出。
5. 所有提取出的媒体文件必须保存到 `/Users/qunwei/Documents/local_agent_work/output/docx_media/<task_name>/`，不要散落在临时目录。
6. 审校/提取报告里要有一个独立小节：**“图片与图注审查”**。
7. 如果图片数量巨大（如测试报告里的数百张截图），无法全部读取时要明确告知用户抽样策略，而不是假装已全部处理。

## Anti-patterns

| 错误做法 | 正确做法 |
|----------|----------|
| “我已经读取了 docx 的全部内容”但实际上没看图片 | 先运行媒体审计，确认是否有图片 |
| 看到图片但认为“不重要”就跳过 | 先识别图片类型，再判断是否与任务相关 |
| 把图片散落在 `/tmp` 或当前目录 | 统一保存到 `output/docx_media/<task_name>/` |
| 只提取图片不读内容 | 对含文字/术语的截图使用视觉能力读取 |
| 图片太多就全部跳过 | 抽样关键图片，并向用户说明范围和原因 |
